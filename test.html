Identificar valores atpicos utilizando la desviacin estndar (SD)

Eliminar valores atpicos utilizando la desviacin estndar (SD) es un enfoque comn en estadstica para identificar y tratar puntos de datos que se desvan significativamente de la media de un conjunto de datos. El mtodo implica calcular la desviacin estndar de los datos y eliminar los valores que estn a una cierta distancia (mltiplos de la desviacin estndar) de la media.

Aqu hay una descripcin bsica de cmo funciona este mtodo:

Calcular la Media y la Desviacin Estndar:

Se calcula la media (promedio) y la desviacin estndar de los datos. Establecer un Umbral:

Se elige un mltiplo de la desviacin estndar como umbral. Comnmente, se utilizan mltiplos como 1, 1.5, 2, o 3, dependiendo de la rigurosidad que se desee aplicar. Identificar y Eliminar Valores Atpicos:

Se identifican los valores que estn ms all del umbral establecido en ambos extremos de la distribucin (por encima y por debajo de la media). Estos valores se consideran atpicos y se eliminan del conjunto de datos. Revisar y Validar:

Despus de eliminar los valores atpicos, es importante revisar el conjunto de datos resultante y considerar si la exclusin de esos valores es justificada. En algunos casos, los valores atpicos pueden contener informacin importante o representar eventos genuinos. Es importante destacar que este mtodo asume que los datos siguen una distribucin normal.

# Calcular la media y la desviacin estndar de la columna 'price' en el DataFrame 'model_data'
mean_price <- mean(model_data$price, na.rm = T)  # Calcular la media, ignorando los valores NA

Eliminar valores atipicos con el metodo IQR

Eliminar valores atpicos con el mtodo IQR (rango intercuartlico) es otra tcnica comn en estadstica. El rango intercuartlico es la diferencia entre el tercer cuartil (Q3) y el primer cuartil (Q1) de un conjunto de datos. Los valores atpicos se identifican y eliminan si estn por debajo de Q1 - 1.5 * IQR o por encima de Q3 + 1.5 IQR. Aqu hay un ejemplo de cmo implementar esto en R:

# Calcular el primer cuartil (Q1)
Q1 <- quantile(model_data$price, 0.25)

# Calcular el tercer cuartil (Q3)
Q3 <- quantile(model_data$price, 0.75)

# Calcular el rango intercuartlico (IQR)
IQR <- Q3 - Q1

# Identificar los valores atpicos utilizando el mtodo IQR
outliers <- model_data$price < (Q1 - 1.5 * IQR) | model_data$price > (Q3 + 1.5 * IQR)

# Crear un nuevo DataFrame sin valores atpicos
model2_no_outliers_IQR_method <- model_data[!outliers, ]

El Mtodo de Desviacin Absoluta de la Mediana (Median Absolute Deviation, MAD) es un enfoque robusto utilizado para identificar valores atpicos en un conjunto de datos. A diferencia de otros mtodos que dependen de la media y la desviacin estndar, el MAD utiliza la mediana como medida central y evala la dispersin de los datos en trminos de desviaciones absolutas respecto a la mediana.

Coeficiente de correlacin

El coeficiente de correlacin es una medida estadstica que cuantifica la relacin lineal entre dos variables. En otras palabras, el coeficiente de correlacin indica en qu medida las variaciones en una variable estn asociadas con las variaciones en otra variable. Esta medida se utiliza comnmente para evaluar la fuerza y direccin de la relacin entre dos variables continuas.

Existen diferentes tipos de coeficientes de correlacin, pero uno de los ms comunes es el coeficiente de correlacin de Pearson. El coeficiente de correlacin de Pearson, denotado como r, tiene un rango de -1 a 1, donde:

  • 1: Indica una correlacin perfecta positiva. Esto significa que a medida que una variable aumenta, la otra variable tambin aumenta en una relacin lineal perfecta.

  • 0: Indica ausencia de correlacin lineal. No hay relacin lineal entre las dos variables.

  • -1: Indica una correlacin perfecta negativa. Esto significa que a medida que una variable aumenta, la otra variable disminuye en una relacin lineal perfecta.

Transformaciones de datos

Las transformaciones de datos, como tomar el logaritmo, la raz cuadrada o elevar al cuadrado, son tcnicas utilizadas en el anlisis de datos por diversas razones. El propsito principal de estas transformaciones es modificar la distribucin de los datos o la relacin entre las variables para satisfacer las suposiciones de los mtodos estadsticos o mejorar la interpretabilidad de los resultados. Aqu hay algunas razones comunes para realizar transformaciones:

  • Estabilizar la Varianza:

En el anlisis de regresin, se asume que la varianza de los errores es constante en todos los niveles de las variables predictoras. Si hay una relacin no constante entre la varianza y las variables predictoras, las transformaciones como la raz cuadrada o el logaritmo pueden ayudar a estabilizar la varianza.

  • Linearizar Relaciones No Lineales:

En algunos casos, la relacin entre las variables puede ser no lineal, pero los modelos estadsticos asumen una relacin lineal. Tomar el logaritmo o la raz cuadrada de una variable puede ayudar a linealizar la relacin, permitiendo as que se apliquen modelos lineales.

  • Manejar Sesgo en la Distribucin:

Si los datos tienen una distribucin sesgada hacia la derecha, donde hay valores atpicos que se encuentran en el extremo superior, tomar el logaritmo puede reducir el sesgo y hacer que la distribucin sea ms simtrica.

  • Escalamiento:

Elevar al cuadrado o tomar la raz cuadrada de una variable puede cambiar la escala de los datos, lo que puede ser til para resaltar patrones o reducir la importancia de valores extremos.

  • Condiciones de Normalidad:

En algunos mtodos estadsticos, como las pruebas de hiptesis paramtricas, se asume la normalidad de los datos. Transformaciones como el logaritmo pueden ayudar a que los datos se aproximen ms a una distribucin normal.

  • Mejorar la Interpretabilidad:

En ocasiones, realizar transformaciones puede hacer que los resultados sean ms interpretables o ms fcilmente comparables. Es importante sealar que la eleccin de una transformacin especfica depende de la naturaleza de los datos y de los objetivos del anlisis. Adems, se debe tener precaucin al interpretar resultados transformados, ya que pueden afectar la interpretacin de los coeficientes o las mtricas de evaluacin del modelo. La seleccin de una transformacin adecuada a menudo implica explorar y comprender la naturaleza de los datos.

Particin de Datos

La particin de datos es una tcnica comn en estadsticas y aprendizaje automtico que implica dividir un conjunto de datos en subconjuntos ms pequeos para diversos propsitos, como entrenar modelos, validar modelos y evaluar su rendimiento. Las particiones de datos son fundamentales para garantizar la validez y la generalizacin de los modelos que se construyen.

# Definir una funcin para dividir los datos en conjuntos de entrenamiento y prueba
split_data_fun <- function(df, var1, var2) {
    
    # Obtener los niveles nicos de las variables var1 y var2
    levels1 <- unique (df[[var1]])
    levels2 <- unique (df[[var2]])
    
    # Inicializar ndices para conjuntos de entrenamiento y prueba
    training_indices <- c()
    testing_indices  <- c()
    
    # Iterar sobre los niveles de var1 y var2
    for (level1 in levels1) {
        for (level2 in levels2) {
            
            # Filtrar las filas que coinciden con los niveles actuales de var1 y var2
            rows <- which(df[[var1]] == level1 & df[[var2]] == level2)
            
            # Verificar si solo hay una instancia
            if (length(rows) < 2 ) {
                # Si solo hay una instancia, aadirla a los datos de entrenamiento
                training_indices <- c(training_indices, rows)
            } else {
                # Si hay ms de una instancia, crear una particin de datos de entrenamiento y prueba
                partition <- createDataPartition(y = rows, p = 0.70, list = FALSE)
                training_indices <- c(training_indices, rows[partition])
                testing_indices <- c(testing_indices, rows[-partition])
            }
        }
    }

    # Devolver una lista con los conjuntos de entrenamiento y prueba
    list(
        training = df[training_indices, ],
        testing = df[testing_indices, ]
    )
}

calc_metrics <- function(dataset, model) {
    
    predictions <- predict(object = model, newdata = dataset)
    actuals     <- dataset$price # Reemplazar 'price' con el nombre de la variable de respuesta
    
    metrics <- list()
    
    # Cacula tus metricas aqui, luego asignalas a lista
    metrics$Model = deparse(substitute (model))
    metrics$MAE   = mae (actual = actuals, predicted = predictions)
    metrics$MSE   = mse (actual = actuals, predicted = predictions)
    metrics$RMSE  = rmse(actual = actuals, predicted = predictions)
    metrics$MAPE  = mape(actual = actuals, predicted = predictions) 
    return(metrics)
}
